# Teaching Notes

## Preferences
- **Intuition-first, light math.** Diagrams and plain-English mechanisms lead;
  equations appear only where they clarify. Occasional runnable (nanoGPT-style) code is welcome.
- Learner already knows MLP/backprop vocabulary — *use it as scaffolding*, don't re-explain it.

## Working notes
- Syllabus source of truth: `index.md` (Tier 1 of a larger AI-engineering path).
- Dual mission: practical (better LLM apps) + mastery (sketch a transformer from memory).
  Every lesson should land at least one practical "why the model behaves like this" payoff.
