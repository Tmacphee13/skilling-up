NGS Foundations · Lesson 2

How the molecule gets read

Flow cell → clusters → cycles → chemistry. What "2×150" and a quality plot really mean.

In Lesson 1 you built the molecule and saw that P5/P7 exist to grab the flow cell. This lesson picks up right there: the molecule is attached — now how does the machine turn it into bases? Three words carry the whole story: flow cell, cluster, cycle.

Win for today: you'll be able to say exactly what a "cycle" is, what an instrument's "chemistry" means, and why the per-cycle quality plot in your QC reports slopes downward.

The flow cell — the stage

A flow cell is a glass slide with fluidic channels (lanes) whose surface is coated with a lawn of two grafted oligos — the exact complements of P5 and P7. That's the docking station from Lesson 1. Two families of flow cell exist, and the difference will come back to bite you in demux:

  • Non-patterned (MiSeq, HiSeq 2500): molecules land wherever they drift; clusters form at random positions and the software maps them.
  • Patterned (NovaSeq, NextSeq, iSeq): the surface is etched with billions of nanowells at fixed positions. Higher density, lower cost — but the fixed wells and the reagent chemistry make these prone to index hopping, which is Lesson 3.

Clustering — why one molecule isn't enough

A single molecule is far too dim to photograph. So before any reading happens, each molecule is copied in place into a cluster: a tight patch of ~1,000+ identical clones, all descended from that one template. One cluster is bright enough to image, and it becomes exactly one read (or one read pair).

On non-patterned cells this copying is bridge amplification — the strand arches over and primes off a neighboring lawn oligo, repeatedly. On patterned cells it's ExAmp, a fast isothermal reaction where the first molecule to seed a nanowell fills it and physically excludes any second molecule (the "excluded-volume" effect), keeping each well monoclonal — one sample per cluster. Hold onto that idea; when it fails, you get index hopping.

The mental model 1 molecule → 1 cluster → 1 read. Everything about yield ("how many reads did I get?") is really "how many monoclonal clusters formed?" Over-loading the flow cell makes clusters merge and unreadable; under-loading wastes capacity. That balance is the wetlab's loading concentration decision.

A cycle = one base

Now the reading. Illumina's sequencing by synthesis (SBS) rebuilds the complementary strand of the insert one base at a time. Each cycle is a tightly repeated four-step loop across every cluster at once:

  1. Flow in all four nucleotides, each carrying a fluorescent dye and a reversible terminator — a cap that blocks the strand from extending further this cycle.
  2. Each cluster incorporates exactly one matching base. The terminator guarantees no runaway — one base, then it stops.
  3. Image the whole flow cell. Each cluster's color = the base it just added.
  4. Chemically cleave off the dye and the terminator, unblocking the strand. Repeat.

So one cycle adds one base to the read. "2×150" means Read 1 is 150 cycles and Read 2 is 150 cycles. Read length is literally the number of SBS cycles. But that raises a fair question — if Read 1, the index reads, and Read 2 all happen on the same cluster, how does the machine switch between them without doing four separate runs? That's the next section.

"Chemistry" = how many images per cycle

When someone says a run used a certain chemistry, a big part of what they mean is how the instrument distinguishes the four bases per cycle — the number of images (channels) it takes:

ChemistryHow a base is calledInstruments
4-channel4 dyes, 4 images — one color per baseHiSeq, MiSeq
2-channel2 images (red/green): A=both, C=red, T=green, G=neitherNextSeq, NovaSeq
1-channel1 dye, 2 images with a chemistry step betweeniSeq

Notice the 2-channel trick: G is the absence of signal. A dead or empty cluster looks exactly like a G. Step through it yourself — watch the two channels light up and a base get called each cycle:

Why this matters at your desk Two things you actually see land here. (1) Because clusters gradually fall out of sync — some strands lag or run ahead, called phasing / pre-phasing — the signal gets noisier every cycle. That is why the per-cycle quality plot in FastQC/MultiQC slopes down toward the end of the read, and why R2 often looks a bit worse than R1. (2) On 2-channel instruments a run of no-signal can be miscalled as G; a poly-G tail at the 3′ end of reads is the classic 2-channel artifact from dead clusters — now you know it's the "G = dark" rule failing, not real sequence.

One cluster, four reads — and how Read 2 happens

Here's the mechanism your questions were circling. A dual-index paired-end run does not do four separate runs. It reads the same cluster four times, back to back, in a fixed order. The trick to grasp: you never physically "aim" cycles at a segment. Each read is defined by which primer you anneal. A primer only sticks to its one matching site, so annealing a different primer means starting SBS from a different place on the molecule. The loop is always the same: anneal a primer → run N cycles → strip off the new strand → repeat.

  1. Read 1 (→ R1, e.g. 150 cyc)
    The Read 1 primer anneals at SP1 and SBS reads the insert. Then the strand you just synthesized is stripped away, leaving the template bare again.
  2. Index 1 Read (→ I1 = i7, ~8–10 cyc)
    A different primer anneals right next to the i7 barcode. A short burst of cycles reads i7. This is what "separate index cycles" means — its own primer, its own small cycle count, physically after Read 1. Strip.
  3. Index 2 Read (→ I2 = i5, ~8–10 cyc)
    Yet another primer reads i5. There's nothing "pointed" at i5 vs i7 — it's just a different primer landing at a different site. Strip.
  4. Paired-end turnaround (no read — rebuild)
    This is the answer to "how do we get R2 if linearization left everything pointing one way?" The template folds over, bridges to the other grafted lawn oligo, and a polymerase copies it into a fresh complementary strand. Then the original template (the one used for Read 1) is cleaved off and washed away. The cluster is now single-stranded again but pointing the opposite direction — exposing the Read 2 primer site that was buried before.
  5. Read 2 (→ R2, e.g. 150 cyc)
    The Read 2 primer anneals at SP2 on this rebuilt strand and SBS reads the insert from the opposite end. Same insert, other direction — that's paired-end.

So R2 is not read off the original molecule and it is not a second run — it's a strand the instrument synthesizes mid-run, specifically so a fresh primer site becomes available. One cluster, one continuous run, four reads.

This is the root of the i5 flip Notice the i5 read can happen either before the turnaround (off the original strand) or after it (off the rebuilt strand). Those two strands are reverse complements, so the two instrument families report i5 in opposite orientations — the Lesson 1 sample-sheet gotcha. It's not a software bug; it's which strand existed when the i5 primer fired.
