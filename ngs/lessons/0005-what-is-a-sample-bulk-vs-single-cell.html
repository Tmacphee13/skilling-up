What is a "sample"? Bulk vs single-cell

The word "sample" quietly means two different things — and single-cell adds a second demux you already know how to reason about.

In Lesson 3 the sample index (i7/i5) was identity: one library, one sample, demux resolves the pool to samples. That's bulk sequencing, and it hides an assumption. Single-cell breaks it — and the way it breaks it turns out to be the demux skill you already have, applied one level deeper.

Bulk: one library = one sample

You take a tube — a tissue, a blood draw, a culture — extract its nucleic acid, and build one library with one sample index. Every cell in that tube is mixed together and averaged. When you demux, the sample index tells you which tube a read came from, and that's the finest identity that exists. Identity has one level: the sample index.

i7 / i5which sample
R2: cDNA / DNA insertthe biology

Single-cell: one library = one sample = thousands of cells

A single-cell run still makes one library with one sample index — but that library now contains thousands of cells, and you want to keep them apart. You can't give each cell its own sample index (you'd need thousands of libraries). Instead, two more barcodes are added inside the read itself, during prep, when each cell is captured in its own droplet:

  • Cell barcode (16 bp) — every molecule from the same cell gets the same barcode, drawn from a fixed whitelist of ~3.5 million known sequences. This says which cell.
  • UMI (12 bp) — a random tag unique to each original molecule, so PCR copies of the same molecule share a UMI. This says which original molecule, letting you collapse duplicates and count molecules, not reads.

The read structure — where every identity lives

Here's the whole thing for a 10x 3′ library. Notice how the reads divide the labor:

I1sample index
R1: cell barcode16 bp — which cell
+ UMI12 bp — which molecule
R2: cDNAthe transcript
I1 — sample index (which library) R1 cell barcode (which cell) R1 UMI (which molecule) R2 cDNA (the biology)

So the files you'd hand off carry three nested levels of identity:

LevelBarcodeLives inResolved by
Which library?sample indexI1 (/I2)BCL Convert (your L3 demux)
Which cell?cell barcodeR1[0:16]Cell Ranger
Which molecule?UMIR1[16:28]Cell Ranger (dedup)
The unlock Cell-barcode demux is the same operation as sample demux — match an observed barcode against a known list, within a small mismatch tolerance, and bin the read. The whitelist is just a "sample sheet" with 3.5 million rows, and it's applied to R1 by Cell Ranger instead of to I1 by BCL Convert. Everything you learned in Lesson 3 — mismatch budgets, ambiguous matches, unknown-barcode diagnostics — transfers directly. Single-cell is just demux within demux.

Why the reads are lopsided

You only need 28 bp of R1 — exactly the cell barcode + UMI — and there's no biology past it, so R1 is short. R2 carries the actual transcript and gets the long read. That's why single-cell run configs look asymmetric, like 28 × 90. Set R1 too short and you can't parse the barcode/UMI at all — a run-config mistake that makes the data unusable, and one worth catching before the run, not after.

Two failure modes worth a name

  • Doublets — two cells captured in one droplet share a cell barcode, looking like one weird hybrid cell.
  • Ambient RNA — free-floating RNA from lysed cells gets a random barcode, smearing background signal into every cell. This is also why single-cell is extra-sensitive to index hopping — another reason these runs should use UDI (L3).

Spatial assays reuse this exact machinery, but the barcode encodes a location on a slide instead of a droplet — "which cell?" becomes "which spot?"
